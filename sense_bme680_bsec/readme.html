Overview

This example demonstrates the usage of BSEC (Bosch Environmental Cluster) library with BME680 environmental sensor. BSEC library allows to utilize all functionality of BME680 including gas measurements to calculate Indoor Air Quality index (IAQ). It also provides advanced compensation features resulting in more precise temperature and humidity readings.

Hardware Requirements

Compatible evaluation boards:

  • RSL10-SENSE-DB-GEVK
    • 3V Lithium CR2032 battery is required for proper operation.
  • RSL10-SENSE-GEVK
    • 3V Lithium CR2032 battery is required for proper operation.
    • It is required to either solder 10-pin debugging header or use 10-pin needle connector to program / debug the board.
    • Debug probe is not included with the board.

Verification

After the program is flashed and running on RSL10-SENSE-DB-GEVK board it will initialize BME680 sensor and BSEC library. Successful initialization will be signaled by on-board LED turning on for half second.

Afterwards the on-board LED will be turned on during each measurement cycle. Measurement cycle takes around 250 ms and is started periodically every 3 seconds. After each measurement cycle is completed new data will be printed to debug console using SEGGER Real Time Terminal.

The output will look like below image if SEGGER RTT Viewer is used to connect to the console. Each line will contain current output value of given virtual sensor with calibration status in brackets where applicable. Calibration status of 0 means that sensor needs calibration, status 3 meaning sensor is fully calibrated.

Notes

Sometimes the firmware in RSL10 cannot be successfully re-flashed, due to the application going into Sleep Mode or resetting continuously (either by design or due to programming error). To circumvent this scenario, a software recovery mode using DIO12 can be implemented with the following steps:

  1. Connect DIO12 to ground by pressing the PB2 push button.
  2. Press the RESET button (this restarts the application, which will pause at the start of its initialization routine).
  3. Re-flash RSL10. After successful re-flashing, disconnect DIO12 from ground, and press the RESET button so that the application will work properly.
